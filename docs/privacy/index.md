# SSP SDK 隐私权政策

**生效日期：2024年1月1日**

## 简介

SSP SDK（"我们"、"我们的"或"SSP"）是一家专注于为移动应用开发者提供广告变现解决方案的技术服务提供商。本隐私权政策旨在向您详细说明 SSP SDK 在为您提供服务过程中会收集哪些数据、收集的目的、如何使用这些数据，以及您享有的相关权利。

**重要提示：** 本隐私权政策是专门为集成 SSP SDK 的应用开发者（即"您"）编写的技术文档。它与您个人应用的隐私政策是独立的文件。当您的应用集成 SSP SDK 时，您有责任：

1. 在您应用的隐私权政策中向最终用户披露 SSP SDK 的数据收集行为；
2. 确保您获得最终用户对 SSP SDK 数据收集行为的有效同意；
3. 遵守您所在司法管辖区关于数据保护和隐私的法律法规。

## 一、信息收集概述

### 1.1 收集方式

我们通过以下方式收集信息：

- **自动收集**：当最终用户的设备安装并运行集成 SSP SDK 的应用时，我们会自动收集相关设备和使用信息。
- **API 调用**：我们会调用第三方服务（如 Google Play 服务）获取特定标识符信息。
- **地理位置查询**：我们会通过 IP 地址查询服务获取用户的地理位置信息。

### 1.2 我们收集的信息类型

#### 1.2.1 设备和应用信息

| 信息类型 | 说明 | 收集目的 |
|---------|------|---------|
| 设备品牌 | 制造商名称（如 Samsung, Xiaomi, Huawei） | 广告定向 |
| 设备型号 | 具体设备型号（如 Pixel 6, MI 11） | 广告定向 |
| 操作系统版本 | Android 系统版本号（如 13, 14） | 兼容性适配 |
| 屏幕尺寸 | 设备屏幕分辨率（宽度 x 高度） | 广告展示优化 |
| 用户代理 (User-Agent) | 浏览器默认用户代理字符串 | 广告内容适配 |
| 语言设置 | 设备首选语言 | 广告定向 |
| 国家/地区 | 根据 IP 或系统设置确定 | 广告定向 |
| 时区 | 设备所在时区 | 广告时段优化 |

#### 1.2.2 广告标识符

| 信息类型 | 说明 | 收集目的 |
|---------|------|---------|
| Google 广告 ID (GAID) | Android 设备的唯一广告标识符 | 广告追踪与归因 |
| Android ID | 设备的 Android ID（当 GAID 不可用时作为备选） | 广告追踪与归因 |

#### 1.2.3 网络连接信息

| 信息类型 | 说明 | 收集目的 |
|---------|------|---------|
| IP 地址 | 设备的网络 IP 地址 | 地理位置定位 |
| 连接类型 | Wi-Fi / 移动网络 / 以太网 | 广告质量优化 |
| 网络运营商 | 移动网络运营商名称 | 广告定向 |

#### 1.2.4 地理位置信息

| 信息类型 | 说明 | 收集目的 |
|---------|------|---------|
| 经纬度坐标 | GPS 或网络定位提供的坐标（需用户授权） | 本地化广告 |
| 城市/地区 | 基于 IP 或定位服务的地理区域 | 广告定向 |

**注意：** 地理位置信息的收集依赖于最终用户的授权许可。如果用户拒绝授予位置权限，我们不会收集精确的经纬度坐标。

#### 1.2.5 广告事件数据

| 信息类型 | 说明 | 收集目的 |
|---------|------|---------|
| 广告请求事件 | 广告加载请求的时间戳和应用信息 | 服务质量分析 |
| 广告展示事件 | 广告实际展示的时间和广告信息 | 展示统计 |
| 广告点击事件 | 用户点击广告的时间和广告信息 | 点击统计 |
| 广告转化事件 | 用户完成预期操作的信息 | 转化归因 |

## 二、信息收集的技术实现

### 2.1 设备信息收集

我们的 SDK 在初始化时（调用 `SspSdk.init()` 方法时）会自动收集设备信息：

```java
// DeviceInfoManager.java 核心收集逻辑
public class DeviceInfoManager {
    // 屏幕信息
    public int deviceW;  // 屏幕宽度
    public int deviceH;  // 屏幕高度

    // 设备基本信息
    public String brand;    // 设备品牌
    public String model;    // 设备型号
    public String osv;      // 系统版本
    public String ua;      // 用户代理

    // 地理位置
    public Double latitude;   // 纬度
    public Double longitude;  // 经度
    public String country;   // 国家
    public String region;    // 地区
    public String city;      // 城市
}
```

### 2.2 广告标识符获取

我们通过 Google Play 服务获取广告标识符：

```java
// GAIDHelper.java
public class GAIDHelper {
    public static void getGAID(Context context, GAIDCallback callback) {
        // 在子线程中调用 Google 官方接口
        AdvertisingIdClient.Info adInfo = AdvertisingIdClient.getAdvertisingIdInfo(context);
        String gaid = adInfo.getId();
        // 检查用户是否限制了广告追踪
        boolean isLAT = adInfo.isLimitAdTrackingEnabled();
    }
}
```

### 2.3 IP 地理位置查询

我们通过第三方 IP 地理位置服务获取用户的大致位置：

- **主服务**：ipapi.co
- **备用服务**：ipwho.is

这些服务返回的信息包括国家、城市、时区、经纬度等。

### 2.4 事件报告机制

我们的 SDK 会将广告相关事件上报至服务器：

```java
// ReportManager.java
public class ReportManager {
    // 事件数据结构
    public class ReportEvent {
        public String event;    // 事件类型: init, request, show, click
        public int status;     // HTTP 状态码
        public String slotId;  // 广告位 ID
        public String appId;   // 应用 ID
        public String adId;    // 广告 ID
        public String adType;  // 广告类型: BANNER, NATIVE, VIDEO
        public long timestamp; // 时间戳
    }
}
```

报告服务器地址：`http://sdk-sg.huiinex.com/report`

## 三、信息使用目的

我们收集的信息仅用于以下目的：

### 3.1 广告投放与优化

- **广告定向**：利用设备信息、地理位置和广告标识符向用户展示相关性更高的广告。
- **广告竞价**：在程序化广告竞价中提供必要的出价信息。
- **频次控制**：防止同一用户看到过多重复广告。
- **反作弊**：检测和防止无效流量和欺诈行为。

### 3.2 服务质量保障

- **性能监控**：分析广告加载速度、展示成功率等指标。
- **故障排查**：当出现技术问题时，收集相关数据用于诊断。
- **统计分析**：汇总广告效果数据，用于生成业务报告。

### 3.3 合规要求

- **法律法规遵守**：按照适用法律的要求保留必要的记录。
- **争议解决**：在必要时使用数据进行法律争议的解决。

## 四、信息共享

### 4.1 我们不会共享的信息

我们承诺：

- 不会将广告标识符与其他可识别个人身份的信息关联。
- 不会出售最终用户的个人信息。
- 不会将收集的信息用于非广告投放以外的其他目的。

### 4.2 可能的共享情形

在以下情况下，我们可能会披露信息：

| 情形 | 说明 |
|------|------|
| 服务提供商 | 与帮助我们提供服务的合作伙伴共享，这些伙伴受保密协议约束 |
| 法律要求 | 根据法律义务、法院命令或政府要求进行披露 |
| 权利保护 | 为保护我们或第三方的权利、财产或安全 |
| 业务转让 | 在合并、收购或资产出售等商业交易中转让 |

### 4.3 第三方服务

我们的 SDK 可能会调用以下第三方服务：

| 服务 | 提供商 | 用途 | 隐私政策 |
|------|--------|------|---------|
| Google Play 服务 | Google LLC | 获取广告标识符 | [Google 隐私政策](https://policies.google.com/privacy) |
| IP 地理位置服务 | ipapi.co / ipwho.is | 确定用户地理位置 | 参见各服务提供商官网 |

## 五、最终用户权利

### 5.1 限制广告追踪

最终用户可以通过以下方式限制广告追踪：

**Android 设备：**

1. 打开设备「设置」
2. 进入「Google」→「广告」
3. 启用「退出广告个性化」

或者：

1. 打开设备「设置」
2. 进入「隐私」→「广告 ID」
3. 启用「限制广告追踪」

### 5.2 重置广告标识符

用户可以随时重置其广告标识符：

1. 打开设备「设置」
2. 进入「Google」→「广告」
3. 点击「重置广告 ID」

### 5.3 数据删除请求

如果您收到最终用户的数据删除请求，请通过本文档末尾的联系方式联系我们。

## 六、数据安全

### 6.1 安全措施

我们采用以下安全措施保护数据：

- **传输加密**：SDK 与服务器之间的通信使用 HTTPS 协议。
- **访问控制**：严格限制对服务器和数据的访问权限。
- **定期审计**：定期进行安全审计和风险评估。

### 6.2 数据泄露应对

如果我们发现数据安全事件，将按照适用法律的要求通知受影响方和监管机构。

## 七、数据保留

### 7.1 保留期限

| 数据类型 | 保留期限 |
|---------|---------|
| 广告事件数据 | 24 个月 |
| 设备信息（匿名化后） | 18 个月 |
| 用户请求删除的数据 | 立即删除 |

### 7.2 自动清理

如果我们连续 2 年未收到某设备的数据，相关数据将被自动从系统中清除。

## 八、儿童隐私

### 8.1 政策立场

我们**不会故意收集** 13 周岁以下儿童的个人信息。我们要求集成 SSP SDK 的应用开发者：

1. 确保其应用不面向 13 周岁以下儿童。
2. 如果应用面向儿童，应在隐私政策中明确说明。
3. 确保获得监护人对数据收集的有效同意。

### 8.2 发现与处理

如果我们发现收集了儿童个人信息，将立即删除相关数据。

## 九、跨境数据传输

### 9.1 数据存储位置

我们的服务器可能位于不同国家和地区。这意味着您的数据可能会被传输到您所在司法管辖区以外的地方。

### 9.2 合规措施

我们确保跨境数据传输符合适用法律的要求：

- 遵守相关数据保护法规（如 GDPR、CCPA）
- 实施适当的安全保障措施
- 在必要时签订标准合同条款

## 十一、开发者义务

### 11.1 隐私政策披露

作为集成 SSP SDK 的开发者，您需要：

1. **独立隐私政策**：维护您自己的隐私权政策。
2. **SDK 数据披露**：在您的隐私政策中明确说明 SSP SDK 的数据收集行为。
3. **同意机制**：确保获得最终用户对数据收集的适当同意。
4. **权利保障**：为用户提供行使隐私权利的渠道。

### 11.2 建议披露文本

您可以在隐私权政策中添加以下参考文本：

> **广告 SDK 说明**
>
> 我们的应用集成了 SSP SDK，用于广告展示和变现。在提供广告服务的过程中，SSP SDK 可能会收集以下信息：
>
> - 设备信息（品牌、型号、屏幕尺寸、操作系统版本）
> - 广告标识符（Google 广告 ID）
> - 网络信息（IP 地址、连接类型）
> - 地理位置信息（需用户授权）
> - 广告交互事件
>
> SSP SDK 收集这些信息的目的是提供和改进广告服务。详情请参阅 [SSP SDK 隐私权政策](https://ssp-sdk.github.io/ssp-sdk-android/privacy-policy)。

### 11.3 最终用户同意流程

我们建议您实施以下同意流程：

1. **首次启动提示**：在应用首次启动时，向用户说明广告数据收集。
2. **选择退出**：提供关闭个性化广告的选项（调用系统设置）。
3. **权限说明**：在请求位置权限时，说明该权限将用于广告定向。
