概述

本指南旨在帮助开发者正确集成 HuiiAd SDK，并确保符合隐私保护法规的要求。本指南作为 HuiiAd SDK 隐私权政策 的补充，提供具体的实施建议和最佳实践。

隐私合规清单

必检项

  • ☑️ 在应用隐私政策中披露 HuiiAd SDK 数据收集
  • ☑️ 实施最终用户同意机制
  • ☑️ 提供用户退出广告追踪的选项
  • ☑️ 未满 13 岁儿童应用需额外处理

推荐项

  • ○ 在权限请求时说明用途
  • ○ 提供应用内隐私设置
  • ○ 记录用户同意状态

隐私政策披露模板

基本披露文本

建议在您的隐私政策中添加以下内容：

广告服务

我们的应用集成了第三方广告 SDK（HuiiAd SDK），用于在应用中展示广告。在提供广告服务的过程中，HuiiAd SDK 可能会收集以下信息：

  • 设备信息：设备品牌、型号、屏幕尺寸、操作系统版本
  • 广告标识符：Google 广告 ID (GAID)
  • 网络信息：IP 地址、连接类型
  • 地理位置：经纬度坐标（需用户授权）
  • 广告交互数据：广告展示、点击等事件

这些信息用于：

  • 提供个性化广告
  • 广告效果统计
  • 防止无效广告流量

您可以通过以下方式限制广告追踪：

  • 设备设置 > Google > 广告 > 退出广告个性化
  • 或联系：privacy@ssp-sdk.com

详情请参阅：HuiiAd SDK 隐私权政策

儿童应用附加要求

如果您的应用面向 13 岁以下儿童，请添加：

儿童隐私

我们不会故意收集 13 岁以下儿童的个人信息。如果您的应用面向儿童，请确保：

  1. 获得监护人的明确同意
  2. 遵守 COPPA（儿童在线隐私保护法）等相关法规
  3. 限制定向广告的使用

权限使用说明

必须声明的权限

<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

可选权限（用于增强广告定向）

<!-- 精确定位 -->
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />

<!-- 粗略定位 -->
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />

<!-- WiFi 状态 -->
<uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />

常见问题

Q: HuiiAd SDK 会收集哪些敏感信息？

A: HuiiAd SDK 会收集：

  • 广告标识符（GAID）
  • IP 地址
  • 设备信息（品牌、型号、屏幕尺寸）
  • 位置信息（需用户授权）

Q: 如何帮助用户退出广告追踪？

A: 引导用户前往系统设置：

  1. 设置 > Google > 广告 > 退出广告个性化

或使用代码打开系统广告设置页面：

Intent intent = new Intent(Settings.ACTION_AD_SETTINGS);
startActivity(intent);

Q: SDK 初始化时需要用户同意吗？

A: 我们建议在 SDK 初始化前获取用户同意。您可以：

  1. 显示隐私说明对话框
  2. 用户同意后再调用 SspSdk.init()
  3. 记录用户的同意状态

Q: 如果用户拒绝同意怎么办？

A: 您可以选择：

  • 不展示广告
  • 展示不基于用户数据的通用广告
  • 记录拒绝状态，但不阻止用户使用应用
